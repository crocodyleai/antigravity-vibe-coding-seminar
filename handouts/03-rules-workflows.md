# 📖 Session 3: 룰, 워크플로우, 스킬

## 🎯 학습 목표

- 룰(Rules)로 AI 행동 제어
- 워크플로우(Workflows)로 반복 작업 자동화
- 스킬(Skills)로 기능 확장

---

## 📏 룰(Rules)

AI의 **기본 행동 방식**을 정의합니다.

### 룰이란?

- AI가 항상 따라야 하는 규칙
- 프로젝트 전체에 적용
- GEMINI.md 또는 별도 설정에서 정의

### 룰 예시

```markdown
## Rules

1. 모든 함수에 type hints를 사용하세요
2. 에러 발생 시 한국어로 설명하세요
3. 코드 변경 전 항상 이유를 설명하세요
4. 테스트 코드도 함께 작성하세요
```

### 좋은 룰 작성법

| ✅ 좋은 예 | ❌ 나쁜 예 |
|-----------|-----------|
| "변수명은 snake_case 사용" | "좋은 코드 작성" |
| "함수는 20줄 이내로" | "짧게 작성" |
| "에러는 try-except로 처리" | "에러 처리 잘 하기" |

## 🔄 워크플로우(Workflows)

**반복 작업을 자동화**하는 명령 시퀀스입니다.

### 워크플로우란?

- 여러 단계의 작업을 하나로 묶음
- 프롬프트 하나로 복잡한 작업 실행
- 일관된 결과 보장

### 워크플로우 예시

```markdown
## Workflows

### /review
코드 리뷰를 수행합니다:
1. 열린 파일의 코드 분석
2. 버그 가능성 체크
3. 성능 개선점 제안
4. 코딩 컨벤션 준수 여부 확인

### /test
테스트를 생성합니다:
1. 현재 함수/클래스 분석
2. pytest 기반 테스트 코드 생성
3. edge case 포함
4. 테스트 실행 및 결과 확인
```

### 사용법

채팅창에 `/review` 또는 `/test` 입력하면 해당 워크플로우 실행!

## 🛠️ 스킬(Skills)

AI의 **능력을 확장**하는 기능 모듈입니다.

### 스킬이란?

- 특정 도메인/작업에 특화된 기능
- 외부 도구/API 연동 가능
- 재사용 가능한 기능 단위

### 스킬 예시

| 스킬 | 기능 |
|------|------|
| 웹 검색 | 최신 정보 검색 |
| 파일 읽기/쓰기 | 프로젝트 파일 조작 |
| 터미널 실행 | 명령어 실행 |
| 이미지 생성 | 다이어그램/차트 생성 |

### Antigravity 기본 스킬

1. **코드 생성** - 자연어 → 코드
2. **코드 설명** - 코드 → 자연어
3. **디버깅** - 에러 분석 및 수정
4. **리팩토링** - 코드 개선

## 🔗 룰 + 워크플로우 + 스킬 조합

```
┌─────────────────────────────────────────┐
│              GEMINI.md                  │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│  │  Rules  │ │Workflows│ │ Skills  │   │
│  │ 행동규칙 │ │자동화    │ │기능확장  │   │
│  └────┬────┘ └────┬────┘ └────┬────┘   │
│       │           │           │         │
│       └───────────┼───────────┘         │
│                   │                     │
│              AI 에이전트                 │
│                   │                     │
│              사용자 요청                 │
└─────────────────────────────────────────┘
```

### 실전 예시

**상황**: 새 API 엔드포인트 개발

1. **룰** 적용: type hints, docstring 자동 포함
2. **워크플로우** `/api` 실행: 
   - 라우터 생성
   - 스키마 정의
   - 테스트 코드 생성
3. **스킬** 활용: 터미널에서 서버 실행, 테스트 수행
